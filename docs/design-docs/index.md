# Design Docs Index

設計判断のカタログです。重要なアーキテクチャ・設計の意思決定はここに記録します。

> **なぜこのカタログが重要か**:
> エージェントの実行時に参照できない知識は存在しないも同然です。
> Slackの議論・口頭での合意・個人の頭の中の知識はここにエンコードしてください。

---

## 検証ステータス

| ステータス | 意味 |
|---|---|
| ✅ Active | 現在有効な設計判断 |
| 🔄 Under Review | 見直し中 |
| ❌ Superseded | 別の決定に置き換えられた（履歴として保持） |

---

## 設計ドキュメント一覧

| ドキュメント | ステータス | 概要 |
|---|---|---|
| [`core-beliefs.md`](./core-beliefs.md) | ✅ Active | エージェントファーストの中核的信念 |

## 新しい設計ドキュメントの作成方法

```markdown
# [設計判断のタイトル]

**ステータス**: Active / Under Review / Superseded
**日付**: YYYY-MM-DD
**決定者**: (人間 or エージェント)

## 背景
<!-- なぜこの決定が必要だったか -->

## 検討した選択肢
<!-- 何と何を比較したか -->

## 決定
<!-- 何を選んだか、なぜか -->

## 影響
<!-- この決定がシステムに与える影響 -->
```
