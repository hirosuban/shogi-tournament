# Product Specs Index

プロダクト仕様書のカタログです。

> プロダクト仕様はエージェントが機能を実装する際の「受け入れ基準」として機能します。
> ユーザーフィードバック・要望は仕様書に変換してからエージェントに渡してください。

---

## 仕様書の書き方

```markdown
# [機能名]

## 概要
<!-- この機能が解決するユーザーの課題 -->

## ユーザーストーリー
As a [ユーザー],
I want to [やりたいこと],
So that [得られる価値].

## 受け入れ基準
- [ ] 条件1
- [ ] 条件2

## スコープ外
<!-- 明示的にやらないこと -->

## デザイン参考
<!-- モックアップ・参考UIなど -->
```

## ステータス定義

| ステータス | 意味 |
|---|---|
| 📝 Draft | 作成中・未確定 |
| ✅ Active | 実装対象として確定済み |
| 🚀 Shipped | リリース済み |
| ❌ Cancelled | 実装しないことに決定 |
