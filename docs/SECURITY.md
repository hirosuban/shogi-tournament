# SECURITY.md

セキュリティ原則と境界バリデーション要件を定義します。

---

## 基本原則

1. **境界でバリデーション** — 外部からの入力はすべて境界でスキーマ検証を行う
2. **最小権限** — 各コンポーネントは必要最小限の権限のみ持つ
3. **秘密情報の保護** — 認証情報・トークン・APIキーはコードにハードコードしない
4. **防御的プログラミング** — 入力値は常に不正であると仮定してバリデーションする

---

## 境界バリデーション要件

以下はすべて境界（APIエンドポイント・外部サービス呼び出し・ファイル読み込み）でバリデーションすること:

- HTTPリクエストのbody・query・params
- 外部APIのレスポンス
- 環境変数
- ファイルシステムからの読み込み

バリデーションにはスキーマライブラリを使用する（例: Zod, Pydantic相当）。
型だけに頼った「型アサーション」は境界では使用しない。

## 秘密情報の管理

- `.env` ファイルはリポジトリにコミットしない
- `.env.example` に必要な変数のキーのみ記載する
- 本番環境の秘密情報は環境変数または専用のシークレット管理サービスで管理する

---

## 依存関係のセキュリティ

- 既知の脆弱性がある依存関係を定期的にチェックする
- 不要な依存関係は積極的に削除する
- 依存関係のバージョンは固定する（lockファイルをコミットする）

## OWASP Top 10 対策

| 脅威 | 対策 |
|---|---|
| インジェクション | パラメータ化クエリ・スキーマバリデーション |
| 認証の不備 | 標準ライブラリを使用・セッション管理を適切に |
| XSS | 出力エスケープ・CSP設定 |
| CSRF | CSRFトークン・SameSite Cookie |
| セキュリティの設定ミス | 本番環境でのデバッグ情報の非表示 |
